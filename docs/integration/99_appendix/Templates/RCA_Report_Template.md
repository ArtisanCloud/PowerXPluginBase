## ⚙️ 四、根因分析（Root Cause）

| 层级 | 根因说明 |
|------|-----------|
| 应用层 | 连接池配置错误（max_idle_conn = 0） |
| 环境层 | Redis 超时配置过低 |
| 网络层 | 宿主代理端口未开放 |
| 外部依赖 | License Server 响应延迟 |

---

## 🧠 五、解决方案与恢复步骤

1️⃣ 修复配置文件 `/config/config.yaml`；  
2️⃣ 扩展数据库连接池；  
3️⃣ 增加 Redis 超时阈值至 10s；  
4️⃣ 部署后执行 `powerx plugin restart`；  
5️⃣ 验证所有租户 License 状态恢复。

## 🔐 六、防止复发的长期措施

| 领域 | 改进措施 |
|------|----------|
| 配置管理 | 引入配置校验工具（lint + CI check） |
| 部署流程 | 增加 pre-check 阶段（DSN / Redis Ping） |
| 监控告警 | 设置指标触发阈值（error_rate > 3%） |
| 测试覆盖 | 增强集成测试（License / Schema / Redis） |
